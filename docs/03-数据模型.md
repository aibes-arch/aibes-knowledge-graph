# 03 数据模型

## 3.1 文档模型

```python
class Document(BaseModel):
    id: str              # UUID
    filename: str        # 原始文件名
    file_type: str       # pdf / docx / markdown / text
    domain: str          # 领域标识，默认 equipment_maintenance
    status: str          # uploaded / parsed / extracted / graph_written
    storage_path: str    # 本地存储路径
    created_at: datetime # 创建时间
```

## 3.2 文档切块模型

```python
class Chunk(BaseModel):
    id: str              # UUID
    document_id: str     # 所属文档 ID
    chunk_index: int     # 块序号
    title: str           # 块标题（可选）
    content: str         # 文本内容
    page_no: int         # 所在页码
    token_count: int     # 字符数/Token 数
```

## 3.3 抽取候选模型

```python
class Candidate(BaseModel):
    id: str              # UUID
    chunk_id: str        # 来源块 ID
    document_id: str     # 来源文档 ID
    entity_json: dict    # {"entities": [...]}
    relation_json: dict  # {"relations": [...]}
    confidence: float    # 置信度
    status: str          # pending / approved / rejected / error
    created_at: datetime
```

其中 `entity_json` 中的实体结构：

```json
{
  "name": "电机轴承",
  "type": "Component",
  "properties": {},
  "evidence": "原文片段",
  "confidence": 0.85
}
```

关系结构：

```json
{
  "source": "电机轴承",
  "target": "异响",
  "type": "HAS_FAULT",
  "properties": {},
  "evidence": "原文片段",
  "confidence": 0.75
}
```

## 3.4 领域 Schema 模型

```python
class EntityType(BaseModel):
    label: str           # 英文类型名，如 Equipment
    name: str            # 中文类型名，如 装备
    color: Optional[str] # 可视化颜色

class RelationType(BaseModel):
    type: str            # 关系类型，如 HAS_COMPONENT
    from_: str           # 源实体类型
    to: str              # 目标实体类型

class DomainSchema(BaseModel):
    domain: str
    entities: List[EntityType]
    relations: List[RelationType]
```

### 默认装备维修 Schema

**实体类型：**

| Label | 中文名 |
|-------|--------|
| Equipment | 装备 |
| Subsystem | 子系统 |
| Component | 部件 |
| Fault | 故障 |
| Symptom | 现象 |
| Cause | 原因 |
| Solution | 维修方案 |
| Procedure | 维修步骤 |
| Tool | 工具 |
| SparePart | 备件 |
| Supplier | 供应商 |
| Document | 文档 |
| Person | 专家 |
| Record | 维修记录 |

**关系类型：**

| 关系 | 源 → 目标 |
|------|-----------|
| HAS_SUBSYSTEM | Equipment → Subsystem |
| HAS_COMPONENT | Subsystem → Component |
| HAS_FAULT | Component → Fault |
| HAS_SYMPTOM | Fault → Symptom |
| CAUSED_BY | Fault → Cause |
| SOLVED_BY | Fault → Solution |
| HAS_STEP | Solution → Procedure |
| REQUIRES_TOOL | Solution → Tool |
| REQUIRES_PART | Solution → SparePart |
| SUPPLIED_BY | SparePart → Supplier |
| MENTIONS | Document → Equipment |
| CONFIRMS | Record → Fault |
| PROVIDES | Person → Solution |

## 3.5 Neo4j 图模型

### 节点

```cypher
(:Entity {
  name: "电机轴承",
  type: "Component",
  confidence: 0.85,
  evidence: "...",
  updatedAt: datetime()
})

(:Document {
  id: "uuid",
  updatedAt: datetime()
})
```

### 关系

```cypher
(:Entity)-[:RELATED {type: "HAS_FAULT", confidence: 0.75}]->(:Entity)
(:Document)-[:MENTIONS]->(:Entity)
```

### 索引

```cypher
CREATE INDEX entity_name_type IF NOT EXISTS FOR (n:Entity) ON (n.name, n.type)
CREATE INDEX entity_type IF NOT EXISTS FOR (n:Entity) ON (n.type)
CREATE INDEX doc_id IF NOT EXISTS FOR (d:Document) ON (d.id)
```
