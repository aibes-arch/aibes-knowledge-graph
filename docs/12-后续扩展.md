# 12 后续扩展

## 12.1 存储层扩展

### PostgreSQL 元数据存储

将文档、切块、候选、审核记录持久化到 PostgreSQL，替代当前内存存储：

```sql
CREATE TABLE documents (...);
CREATE TABLE chunks (...);
CREATE TABLE extraction_candidates (...);
CREATE TABLE reviews (...);
```

### MinIO 文件存储

将上传的 PDF/Word 等原始文件存入 MinIO，支持大文件、版本管理、预签名访问。

### Qdrant 向量库

为每个文本块生成 Embedding 并存储到 Qdrant，支持：

- 语义相似检索
- 混合检索（向量 + 图谱）
- 实体 Embedding 相似去重

## 12.2 任务队列扩展

### Celery + Redis

将耗时操作转为异步任务：

- 文档解析
- LLM 抽取
- 图写入
- 批量处理

### Flower 监控

可视化任务队列、重试、失败处理。

## 12.3 实体对齐与去重

### 多层对齐策略

```text
规则层：大小写、空格、符号归一化
词典层：同义词库、别名库
向量层：Embedding 相似度 > 0.88
LLM 层：让模型判断是否为同一实体
人工层：低置信度进入审核队列
```

### 对齐效果

将「风机 / 冷却风机 / Cooling Fan / 散热风扇 / Fan Assembly」归一为同一实体。

## 12.4 人工审核中心

- 查看待审核候选
- 批量通过 / 拒绝
- 编辑实体名称、类型、关系
- 导出审核记录

## 12.5 Schema 配置器

可视化 Schema 设计器：

- 添加/删除实体类型
- 添加/删除关系类型
- 设置实体颜色
- 导入/导出 Schema JSON
- 多领域 Schema 切换

## 12.6 GraphRAG 增强

参考 Microsoft GraphRAG：

1. 从文本抽取知识图谱
2. 社区检测（Leiden / Louvain）
3. 生成社区摘要
4. 全局性问题利用社区摘要回答
5. 局部性问题利用多跳子图回答

## 12.7 OCR 与多模态

- 引入 PaddleOCR / MinerU / Docling
- 支持扫描版 PDF
- 支持图片中的表格、公式识别
- 支持图文混合文档

## 12.8 业务智能体

基于图谱构建垂直领域 Agent：

| Agent | 能力 |
|-------|------|
| 故障诊断 Agent | 根据现象推理故障原因 |
| 维修指导 Agent | 推荐维修方案与步骤 |
| 备件推荐 Agent | 推荐所需备件与供应商 |
| 报表生成 Agent | 自动生成知识统计报表 |
| 合规审查 Agent | 检查文档完整性与一致性 |
