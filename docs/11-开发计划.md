# 11 开发计划

## 11.1 MVP 范围（当前版本 v0.1.0）

目标：实现最核心链路，验证从技术到产品的可行性。

### 已实现功能

- [x] 上传 PDF / Word / Markdown / TXT
- [x] 自动解析文本并切块
- [x] 自动抽取实体关系（LLM + Mock）
- [x] 写入 Neo4j
- [x] 图谱可视化 + 问答

### 未实现（故意不做）

- [ ] Celery 异步队列
- [ ] PostgreSQL / Qdrant / MinIO
- [ ] 实体对齐与去重
- [ ] 人工审核中心
- [ ] 多领域 Schema 配置 UI
- [ ] 用户权限管理

## 11.2 迭代路线

### 第一阶段：MVP 验证（已完成）

- 确认技术栈可行性
- 验证 LLM 抽取效果
- 验证 Neo4j 写入与查询
- 验证前端可视化

### 第二阶段：数据持久化（v0.2.0）

- 引入 PostgreSQL 存储文档/切块/候选
- 引入 MinIO 存储原始文件
- 引入 Qdrant 存储文本向量
- 实现文档管理列表

### 第三阶段：异步处理（v0.3.0）

- 引入 Celery + Redis
- 文档解析、抽取、写入图谱改为异步任务
- 增加任务状态监控
- 支持批量上传

### 第四阶段：知识质量（v0.4.0）

- 实现实体对齐与去重
- 引入同义词库
- 低置信度候选进入人工审核
- 审核中心 UI

### 第五阶段：Schema 配置（v0.5.0）

- Schema 设计器界面
- 多领域 Schema 切换
- 自定义实体颜色、关系约束

### 第六阶段：GraphRAG 增强（v0.6.0）

- 社区检测与社区摘要
- 向量检索 + 图检索混合 RAG
- 答案引用原文片段与图谱路径

### 第七阶段：业务智能体（v0.7.0+）

- 故障诊断 Agent
- 维修指导 Agent
- 备件推荐 Agent
- 报表生成 Agent

## 11.4 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| LLM 抽取成本高 | 高 | 支持本地模型（Ollama/vLLM）、批量抽取 |
| 实体对齐效果差 | 中 | 结合规则、Embedding、LLM 多层判断 |
| 大文档解析慢 | 中 | 异步任务、分布式解析 |
| Neo4j 性能瓶颈 | 中 | 索引优化、分片、集群 |
