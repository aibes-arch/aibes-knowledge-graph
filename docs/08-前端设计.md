# 08 前端设计

## 8.1 设计原则

- **简单直接**：MVP 阶段使用原生 HTML + JS，无需构建工具
- **三栏布局**：左侧操作面板、中间图谱、右侧问答与详情
- **实时反馈**：每个操作后显示状态与结果


## 8.4 交互流程

1. 用户选择文件，点击「上传文档」
2. 点击「解析文本」生成文本块
3. 点击「抽取知识」调用 LLM/Mock 抽取
4. 点击「写入图谱」将数据写入 Neo4j
5. 点击「加载全图」或「搜索节点」查看图谱
6. 点击节点查看详情
7. 在右侧输入问题，点击「提问」获取 GraphRAG 回答

## 8.5 可视化设计

### 节点

- 圆形节点，半径 18px
- 颜色按实体类型区分：

| 类型 | 颜色 |
|------|------|
| Equipment | #e53935 |
| Component | #8e24aa |
| Fault | #fb8c00 |
| Solution | #1e88e5 |
| Tool | #6d4c41 |
| SparePart | #78909c |
| Document | #5e35b1 |
| 其他 | #607d8b |

### 关系

- 灰色连线，透明度 0.6
- 连线中部显示关系类型标签

### 力导向模拟

```javascript
simulation = d3.forceSimulation(nodes)
  .force("link", d3.forceLink(links).distance(120))
  .force("charge", d3.forceManyBody().strength(-400))
  .force("center", d3.forceCenter(width / 2, height / 2))
  .force("collide", d3.forceCollide().radius(35));
```

### 交互

- 拖拽节点
- 滚轮缩放
- 点击节点显示详情
- 图例显示当前图中实体类型
