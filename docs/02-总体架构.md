## 2.3 模块划分

### 后端模块

| 模块 | 路径 | 职责 |
|------|------|------|
| API 层 | `backend/app/api/` | 暴露 RESTful 接口 |
| 核心层 | `backend/app/core/` | 配置、LLM 客户端 |
| 服务层 | `backend/app/services/` | 文档解析、切块、抽取、图写入 |
| 模型层 | `backend/app/models/` | Pydantic 数据模型 |

### 前端模块

| 文件 | 职责 |
|------|------|
| `frontend/index.html` | 页面结构与布局 |
| `frontend/app.js` | 交互逻辑、API 调用、D3.js 图谱可视化 |

## 2.4 数据流

1. 用户通过前端上传文档
2. 后端保存文件到 `uploads/` 目录
3. 调用解析服务提取纯文本
4. 调用切块服务将文本切分为语义块
5. 调用抽取服务（LLM 或 Mock）生成候选实体/关系
6. 用户触发写入图谱，后端将候选数据 MERGE 到 Neo4j
7. 前端调用图谱查询接口，使用 D3.js 渲染
8. 用户在问答框输入问题，后端基于图谱上下文调用 LLM 生成答案
