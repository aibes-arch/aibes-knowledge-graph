# AIBES Domain Graph Builder 系统设计说明书

## 版本信息

| 项目 | 内容 |
|------|------|
| 产品名称 | AIBES Domain Graph Builder |
| 版本 | v0.1.0（MVP） |
| 编写日期 | 2026-07-05 |
| 目标 | 面向任意行业的知识图谱自动构建与 GraphRAG 应用开发平台 |

## 1. 项目背景

在装备维修、智能钻井、法律、医疗、制造、科研等领域，大量知识以非结构化形式存在于 PDF、Word、维修记录、专家笔记中。传统检索难以发现深层的因果关系和推理路径。

本系统通过 LLM 自动从文档中抽取实体、关系、属性，构建领域知识图谱，并基于图谱提供增强检索（GraphRAG）和智能问答能力。

## 2. 核心目标

1. **自动化**：上传资料即可自动抽取知识
2. **通用性**：支持多领域 Schema 配置，不写死实体类型
3. **可视化**：直观展示知识网络
4. **可问答**：基于图谱进行语义问答
5. **可扩展**：便于后续接入异步队列、向量库、更多数据源

## 3. 系统架构

```text
数据源层
PDF / Word / Excel / 图片 / 数据库 / API / 网页 / 维修记录 / 专家笔记
        │
        ▼
数据接入层
上传、爬取、同步、OCR、文档解析
        │
        ▼
知识抽取层
分块、实体识别、关系抽取、属性抽取、事件抽取
        │
        ▼
知识融合层
实体对齐、去重、同义词归一、冲突检测、人工审核
        │
        ▼
存储层
Neo4j 图数据库 + PostgreSQL 元数据 + MinIO 文件库 + Qdrant 向量库
        │
        ▼
服务层
图谱查询、GraphRAG、知识问答、可视化、API
        │
        ▼
应用层
知识检索、故障诊断、维修指导、备件推荐、报表生成、智能助手
```

## 4. MVP 范围

第一版仅实现 5 个功能：

1. 上传 PDF / Word / Markdown / TXT
2. 自动解析文本
3. 自动抽取实体关系
4. 写入 Neo4j
5. 图谱可视化 + 问答

## 5. 关键技术

| 层级 | 技术 |
|------|------|
| 后端 | Python FastAPI |
| 文档解析 | pdfplumber / python-docx / markdown |
| LLM | OpenAI-compatible API（支持 DeepSeek / Qwen / vLLM / Ollama） |
| 图数据库 | Neo4j |
| 向量库 | Qdrant（预留） |
| 关系库 | PostgreSQL（预留） |
| 文件库 | MinIO（预留） |
| 任务队列 | Celery + Redis（预留） |
| 前端 | HTML + D3.js |
| 部署 | Docker Compose |

## 6. 默认领域 Schema

本 MVP 预置「装备维修」领域 Schema。

实体类型：Equipment, Subsystem, Component, Fault, Symptom, Cause, Solution, Procedure, Tool, SparePart, Supplier, Document, Person, Record

核心关系：

```text
装备 HAS_SUBSYSTEM 子系统
子系统 HAS_COMPONENT 部件
部件 HAS_FAULT 故障
故障 HAS_SYMPTOM 现象
故障 CAUSED_BY 原因
故障 SOLVED_BY 维修方案
维修方案 HAS_STEP 维修步骤
维修方案 REQUIRES_TOOL 工具
维修方案 REQUIRES_PART 备件
备件 SUPPLIED_BY 供应商
文档 MENTIONS 实体
维修记录 CONFIRMS 故障
专家 PROVIDES 经验
```

## 7. 接口总览

| 路径 | 方法 | 说明 |
|------|------|------|
| `/documents/upload` | POST | 上传文档 |
| `/documents/{id}/parse` | POST | 解析文档 |
| `/documents/{id}/extract` | POST | 抽取知识 |
| `/documents/{id}/write-graph` | POST | 写入图谱 |
| `/graph/search` | GET | 搜索节点 |
| `/graph/neighbors` | GET | 邻居查询 |
| `/graph/full` | GET | 全图查询 |
| `/rag/chat` | POST | 知识问答 |
| `/schema/default` | GET | 默认 Schema |

## 8. 部署方式

- 本地开发：Python venv + 本地 Neo4j
- Docker Compose：一键启动 Neo4j + Backend
- 生产：Gunicorn + Nginx + Neo4j 集群

## 9. 使用流程

1. 上传文档
2. 解析文本
3. 抽取知识
4. 写入图谱
5. 查看可视化图谱
6. 进行知识问答
