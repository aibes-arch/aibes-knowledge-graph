# 05 接口设计

## 5.1 接口概览

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/` | 服务信息 |
| GET | `/docs` | Swagger 文档 |
| GET | `/schema/default` | 获取默认领域 Schema |
| POST | `/schema` | 创建/提交 Schema |
| POST | `/documents/upload` | 上传文档 |
| POST | `/documents/{id}/parse` | 解析文档 |
| POST | `/documents/{id}/extract` | 抽取知识 |
| POST | `/documents/{id}/write-graph` | 写入 Neo4j |
| GET | `/documents/{id}` | 获取文档信息 |
| GET | `/documents/{id}/candidates` | 获取抽取候选 |
| GET | `/graph/statistics` | 图谱统计 |
| GET | `/graph/search` | 搜索节点 |
| GET | `/graph/neighbors` | 获取节点邻居 |
| GET | `/graph/full` | 获取全图 |
| POST | `/rag/chat` | 图谱增强问答 |

## 5.2 详细接口说明

### 5.2.1 获取默认 Schema

```http
GET /schema/default
```

**响应示例：**

```json
{
  "domain": "equipment_maintenance",
  "entities": [
    {"label": "Equipment", "name": "装备", "color": null},
    {"label": "Component", "name": "部件", "color": null}
  ],
  "relations": [
    {"type": "HAS_COMPONENT", "from": "Equipment", "to": "Component"}
  ]
}
```

### 5.2.2 上传文档

```http
POST /documents/upload
Content-Type: multipart/form-data
```

**请求参数：**

| 字段 | 类型 | 说明 |
|------|------|------|
| file | File | PDF / DOCX / MD / TXT |

**响应示例：**

```json
{
  "document_id": "uuid",
  "filename": "sample.md",
  "status": "uploaded"
}
```

### 5.2.3 解析文档

```http
POST /documents/{id}/parse
```

**响应示例：**

```json
{
  "document_id": "uuid",
  "status": "parsed",
  "text_length": 216,
  "chunks": 1
}
```

### 5.2.4 抽取知识

```http
POST /documents/{id}/extract
```

**响应示例：**

```json
{
  "document_id": "uuid",
  "status": "extracted",
  "candidates": 3
}
```

### 5.2.5 写入图谱

```http
POST /documents/{id}/write-graph
```

**响应示例：**

```json
{
  "document_id": "uuid",
  "status": "graph_written"
}
```

### 5.2.6 获取文档候选

```http
GET /documents/{id}/candidates
```

**响应示例：**

```json
[
  {
    "id": "uuid",
    "chunk_id": "uuid",
    "document_id": "uuid",
    "entity_json": {"entities": [...]},
    "relation_json": {"relations": [...]},
    "confidence": 0.8,
    "status": "pending"
  }
]
```

### 5.2.7 图谱统计

```http
GET /graph/statistics
```

**响应示例：**

```json
{
  "entity_count": 25,
  "relation_count": 38,
  "document_count": 1
}
```

### 5.2.8 搜索节点

```http
GET /graph/search?keyword=风机&limit=50
```

**响应示例：**

```json
{
  "nodes": [
    {"name": "风机", "type": "Equipment", ...}
  ]
}
```

### 5.2.9 获取邻居

```http
GET /graph/neighbors?name=风机&type=Equipment&depth=2
```

**响应示例：**

```json
[
  {
    "nodes": [...],
    "relationships": [...]
  }
]
```

### 5.2.10 获取全图

```http
GET /graph/full?limit=300
```

**响应示例：**

```json
{
  "nodes": [...],
  "links": [...]
}
```

### 5.2.11 图谱增强问答

```http
POST /rag/chat
Content-Type: application/json
```

**请求体：**

```json
{
  "question": "风机异响如何处理？"
}
```

**响应示例：**

```json
{
  "question": "风机异响如何处理？",
  "answer": "风机异响通常由电机轴承磨损导致，建议...",
  "sources": [
    {"name": "电机轴承", "type": "Component"},
    {"name": "异响", "type": "Fault"}
  ]
}
```

## 5.3 错误码

| 状态码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误，如文档未解析就抽取 |
| 404 | 文档不存在 |
| 500 | 服务器内部错误，如 Neo4j 连接失败 |
