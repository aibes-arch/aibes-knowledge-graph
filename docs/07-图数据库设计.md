# 07 图数据库设计

## 7.1 数据库选型

采用 **Neo4j** 作为图数据库，原因：

- 原生图存储，适合多跳关系查询
- Cypher 查询语言表达力强
- 与 LLM/GraphRAG 生态结合紧密
- 提供可视化工具（Neo4j Browser）

## 7.2 节点设计

### Entity 节点

```cypher
(:Entity {
  name: "电机轴承",
  type: "Component",
  confidence: 0.85,
  evidence: "冷却风机出现异响，可能是电机轴承磨损导致",
  updatedAt: datetime()
})
```

### Document 节点

```cypher
(:Document {
  id: "uuid",
  updatedAt: datetime()
})
```

## 7.3 关系设计

### 实体间关系

```cypher
(:Entity)-[:RELATED {type: "HAS_FAULT", confidence: 0.75}]->(:Entity)
```

使用统一的 `RELATED` 关系类型，实际关系语义存储在 `type` 属性中。这种设计便于：

- 统一查询所有关系
- 动态扩展关系类型而无需改模型
- 简化可视化渲染

### 文档与实体关系

```cypher
(:Document)-[:MENTIONS]->(:Entity)
```

表示某文档提及了某个实体。

## 7.4 索引设计

```cypher
CREATE INDEX entity_name_type IF NOT EXISTS
FOR (n:Entity) ON (n.name, n.type)

CREATE INDEX entity_type IF NOT EXISTS
FOR (n:Entity) ON (n.type)

CREATE INDEX doc_id IF NOT EXISTS
FOR (d:Document) ON (d.id)
```

## 7.5 核心 Cypher 查询

### 合并实体

```cypher
MERGE (n:Entity {name: $name, type: $type})
SET n += $properties
SET n.confidence = $confidence
SET n.evidence = $evidence
SET n.updatedAt = datetime()
```

### 合并关系

```cypher
MATCH (a:Entity {name: $source})
MATCH (b:Entity {name: $target})
MERGE (a)-[r:RELATED {type: $type}]->(b)
SET r += $properties
SET r.confidence = $confidence
SET r.evidence = $evidence
SET r.updatedAt = datetime()
```

### 搜索节点

```cypher
MATCH (n:Entity)
WHERE n.name CONTAINS $keyword OR n.evidence CONTAINS $keyword
RETURN n {.*} AS node
LIMIT $limit
```

### 邻居查询

```cypher
MATCH path = (n:Entity {name: $name, type: $type})-[:RELATED|MENTIONS*1..2]-(m)
RETURN nodes(path), relationships(path)
LIMIT 100
```

### 全图查询

```cypher
MATCH (n:Entity) RETURN n {.*} AS node LIMIT $limit
MATCH (a:Entity)-[r:RELATED]->(b:Entity)
RETURN {source: a.name, target: b.name, type: r.type} AS rel LIMIT $limit
```

### 统计

```cypher
MATCH (n:Entity) RETURN count(n) AS entity_count
MATCH ()-[r:RELATED]->() RETURN count(r) AS relation_count
MATCH (d:Document) RETURN count(d) AS document_count
```
