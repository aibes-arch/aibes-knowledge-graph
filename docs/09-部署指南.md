# 09 部署指南

## 9.1 本地开发部署

### 前置要求

- Python 3.11+
- Neo4j 5.x（本地或 Docker）
- OpenAI-compatible LLM API Key（可选，可用 MOCK_LLM）

### 步骤 1：克隆项目

```bash
cd aibes-knowledge-graph
```

### 步骤 2：配置环境变量

```bash
cp .env.example .env
```

编辑 `.env`：

```env
LLM_BASE_URL=https://api.openai.com/v1
LLM_API_KEY=sk-your-key
LLM_MODEL=gpt-4o-mini

# 或演示模式
MOCK_LLM=true

NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=password
```

### 步骤 3：启动 Neo4j

#### 方式 A：Docker

```bash
docker compose up -d neo4j
```

#### 方式 B：本地安装

下载 Neo4j Desktop 或 Community Server，设置用户名密码。

### 步骤 4：安装后端依赖

```bash
cd backend
python -m venv venv
venv\Scripts\activate      # Windows
# source venv/bin/activate # macOS/Linux
pip install -r requirements.txt
```

### 步骤 5：启动后端

```bash
uvicorn app.main:app --reload
```

访问：
- API 文档：`http://localhost:8000/docs`
- 服务根地址：`http://localhost:8000/`

### 步骤 6：打开前端

```bash
# 方式一：直接打开文件
start frontend/index.html

# 方式二：通过后端静态服务
# 访问 http://localhost:8000/static/index.html
```

## 9.2 Docker Compose 部署

启动全部服务：

```bash
docker compose up -d
```

服务说明：

| 服务 | 端口 | 说明 |
|------|------|------|
| neo4j | 7474 / 7687 | 图数据库 |
| backend | 8000 | FastAPI 后端 |

访问：
- Neo4j Browser：`http://localhost:7474`
- 后端 API：`http://localhost:8000`
- 前端：`http://localhost:8000/static/index.html`

停止服务：

```bash
docker compose down
```

## 9.3 生产环境部署建议

### 后端

- 使用 Gunicorn + Uvicorn Worker：

```bash
gunicorn app.main:app -w 4 -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:8000
```

- 配置反向代理（Nginx / Traefik）
- 启用 HTTPS
- 配置日志轮转

### 数据库

- Neo4j 集群部署（Enterprise Edition）
- PostgreSQL 持久化元数据
- Qdrant 向量库集群
- MinIO 对象存储集群

### 任务队列

- Celery Worker 多实例
- Redis 作为 Broker 和 Backend
- 配置 Flower 监控任务

### 安全

- API 鉴权（OAuth2 / JWT）
- 文件上传类型与大小限制
- Neo4j 访问控制
- 环境变量加密管理

## 9.4 常见问题

### Q1：Neo4j 连接失败

检查：
- Neo4j 是否启动
- `NEO4J_URI` 配置是否正确
- 防火墙是否放行 7687 端口

### Q2：LLM 调用失败

检查：
- `LLM_API_KEY` 是否有效
- `LLM_BASE_URL` 是否可访问
- 或设置 `MOCK_LLM=true`

### Q3：上传文件失败

检查：
- `uploads/` 目录是否存在且可写
- 文件大小是否超过 `max_upload_size`
